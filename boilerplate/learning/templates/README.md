# Frontend

Static web client for the Django learning API. Served from the same origin as the backend so session cookies work without CORS configuration.

## Pages

| URL | Description |
|-----|-------------|
| `/` | Landing — guest (about + login + Django admin) or authenticated hub |
| `/login/` | Email/password sign-in |
| `/notes/` | Note CRUD, filters, pagination (default: latest 3 notes) |
| `/panel/` | Admin: users, groups, idempotency toggle |
| `/account/` | Password update, deactivate, delete account |
| `/logout/` | Post-logout confirmation page |

## Environment variables

Copy `.env.example` to `.env` at the repo root and adjust values. Django loads it automatically via `python-dotenv`.

| Variable | Purpose |
|----------|---------|
| `API_BASE_URL` | Frontend API origin (empty = same-origin) |
| `SITE_BASE_URL` | Public site URL |
| `ADMIN_URL` | Django admin path |
| `DB_*` | MySQL connection |
| `REDIS_URL` | Redis cache endpoint |

The browser loads `/config.js`, which is generated from these values at runtime.

## Local development

1. Start Django: `python manage.py runserver` from `boilerplate/`
2. Open `http://127.0.0.1:8000/`

Static assets (CSS/JS) are served at `/static/…`.

## Separate deployment

If you host the HTML/JS on a different domain, set the API origin before other scripts load:

```html
<script>
  window.APP_CONFIG = { apiBase: 'https://api.yoursite.com', adminUrl: 'https://api.yoursite.com/admin/' };
</script>
```

You must also configure Django `CSRF_TRUSTED_ORIGINS`, `SESSION_COOKIE_SAMESITE`, and CORS with credentials for cross-origin session auth.

## Idempotency

All note write requests (`POST`, `PUT`, `DELETE`, archive) send a unique `Idempotency-Key` header generated by `js/idempotency.js`. When the backend idempotency flag is disabled, keys are ignored.
