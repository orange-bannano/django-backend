boilerplate\manage.py -> entry point for Django commands (runserver, migrate, test)
boilerplate\boilerplate\__init__.py -> marks the project module as a package
boilerplate\boilerplate\settings.py -> global settings (apps, middleware, database)
boilerplate\boilerplate\urls.py -> top-level URL router
boilerplate\boilerplate\wsgi.py -> WSGI entry point for sync servers
boilerplate\boilerplate\asgi.py -> ASGI entry point for async servers
boilerplate\db.sqlite3 -> local SQLite database (dev only)

boilerplate\learning\__init__.py -> app package marker
boilerplate\learning\apps.py -> app configuration
boilerplate\learning\models.py -> database models, like entities in Spring
boilerplate\learning\admin.py -> admin registrations
boilerplate\learning\utils.py -> safe JSON and pagination helpers
boilerplate\learning\validators.py -> input validation helpers
boilerplate\learning\services.py -> transactional business logic
boilerplate\learning\views.py -> HTTP handlers, requests and response, not like views in Spring, but more like controllers. UI is facilated by templates.
boilerplate\learning\urls.py -> app-level routes
boilerplate\learning\tests.py -> API tests
boilerplate\learning\migrations\ -> database schema history
